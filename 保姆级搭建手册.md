# 📚 三体联动系统：保姆级搭建手册 (SQLite + GUI 进阶版)

> **版本：V2.0 (The SQL Era)**
> **架构：JHenTai (下载) ⟷ Komga (渲染) ⟷ Calibre-Web (书架)**

---

## 🛠️ 0. 核心架构逻辑
本系统通过 **Python + SQLite** 作为数据胶水，将三个独立软件联结为一体：
1.  **JHenTai**：根基。负责连接 E-Hentai 下载并管理本地原始归档。
2.  **Komga**：心脏。提供流媒体级的看漫性能和优秀的 Web 阅读器。
3.  **Calibre-Web**：脸面。负责精美的书架展示、汉化标签搜索与分类。
4.  **中控网关 (Gateway)**：实现从 Calibre 详情页到 Komga 阅读器的毫秒级跳转。

---

## ⚙️ 1. 环境安装清单
在新服务器/机器上，请按序安装以下组件：

* **Python 3.12+**：核心运行环境（务必确保已加入系统 PATH）。
* **Komga (Windows 版)**：建议安装为服务或保持后台运行。
* **Calibre-Web (CPS)**：建议使用带有完整 Python 环境的发行版。
* **Nginx for Windows**：作为流量分发与反向代理的中枢。
* **7-Zip**：用于脚本自动执行 CBZ 封装（需确保 `7z` 命令在 PATH 中）。
* **JHenTai**：下载中枢，务必开启其本地 SQLite 数据库记录功能。

---

## 📂 2. 目录结构规范 (建议维持 Z: 盘符)
为了脚本路径的“无缝迁移”，建议保持以下结构：
* `Z:\comic_tools\`：项目根目录。
    * `scripts\`：存放 GUI、同步、维护等所有 Python 脚本。
    * `configs\`：存放 `config.py` 配置文件。
    * `library\`：**核心数据区**。
        * `sync_master.db`：三体联动核心数据库（替代旧版 JSON）。
        * `backup\`：自动生成的数据库历史备份。
* `Z:\Comics\`：JHenTai 原始下载目录。
* `Z:\Komga\`：脚本生成的规范化 CBZ 库（供 Komga 扫描）。
* `Z:\CalibreLibrary\`：Calibre 数据库文件夹。

## 🔧 3. 关键服务配置

### 3.1 Komga 的路径锁定
必须让 Komga 运行在 `/komga` 路径下，以配合网关跳转。
* **操作**：在 `C:\Users\用户名\AppData\Local\Komga\` 下新建 `application.yml`。
* **内容**：
    ```yaml
    server:
      servlet:
        context-path: /komga
    ```

### 3.2 Nginx 流量中枢 (反向代理)
编辑 `nginx.conf`，确保 `server` 块如下配置（实现 Iframe 嵌入与跨域通行）：
```nginx
server {
    listen 80;
    server_name 192.168.10.102; # 替换为你的服务器内网 IP

    # 1. 任意门网关
    location /jump {
        proxy_pass http://127.0.0.1:8085; 
    }

    # 2. Komga 阅读器 (允许被 Iframe 嵌入)
    location /komga {
        proxy_pass http://127.0.0.1:25600;
        proxy_set_header Host $host;
        proxy_hide_header X-Frame-Options; # 关键：解除 Iframe 限制
        # add_header X-Frame-Options "SAMEORIGIN"; 如果是纯内网环境可忽略，公网建议限制同源
        proxy_cookie_path /komga /komga;
    }

    # 3. Calibre-Web 界面
    location / {
        proxy_pass http://127.0.0.1:8083;
    }
}
```

## 🚀 4. 自动化脚本与中控

### 4.1 `configs/config.py`：总控中心
部署后第一时间修改此文件，确保 IP、账号密码及 **`SYNC_DB_PATH`** 正确。

### 4.2 GUI 中控台 (`GUIManager.py`)
这是你的日常操作界面，通过它管理所有任务：
* **后台常驻**：一键开启“网关”与“巡逻”服务。
* **增量更新**：提供“全自动流水线”和“半自动指引”模式。
* **DEBUG 模式**：独立调用单个脚本进行排查。

---

## 🔄 5. 日常操作流程 (SOP)
当你通过 JHenTai 下载了新漫画后：

1.  **启动中控**：运行 `GUIManager.py`。
2.  **执行同步流水线**：
    * **步骤 1 (巡逻)**：自动初始化数据库并检查异常文件夹。
    * **步骤 2 (J2K)**：自动抓取 JHenTai 元数据、汉化标签并打包 CBZ。
    * **步骤 3 (K2C)**：自动将 CBZ 注入 Calibre-Web 并回填 `calibre_id`。
    * **步骤 4 (Fetch)**：自动指令 Komga 扫盘并抓取 `komga_id`。
3.  **闭环完成**：此时打开 Calibre-Web，点击“在线阅读”，即可直接进入 Komga 的丝滑阅读界面。

---

## 🛡️ 6. 安全与容错机制
* **自动备份**：系统在每次执行巡逻时，会自动在 `library/backup` 生成带时间戳的 `.db` 快照。**如果数据库写挂了，直接从备份恢复即可。**
* **原子更新**：采用 SQL 事务更新，打包成功一本书即写入一次，中途崩溃不会丢失已处理的数据。
* **中断保护**：GUI 提供红色“中断任务”按钮，支持紧急避险，防止错误的数据库操作。
